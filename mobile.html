📦 APK — Android Package Kit

Un APK est une archive (comme un ZIP) contenant toute l'application Android. Il contient :

AndroidManifest.xml
→ Permissions demandées, composants
classes.dex
→ Le code Java/Kotlin compilé
resources.arsc
→ Ressources (images, strings)
META-INF/
→ Signature numérique de l'app
lib/
→ Bibliothèques natives (.so)
⚠️ APK malveillants

Les APK hors Play Store (sideloading) peuvent contenir des malwares. Des groupes Telegram distribuent des APK payants crackés contenant des backdoors, spywares ou ransomwares.

✅ Vérification : SHA-256 sum du fichier APK. Si elle diffère de l'officielle → APK modifié !

Standard mondial

OWASP Top 10

L'OWASP (Open Web Application Security Project) publie les 10 vulnérabilités web les plus critiques. Clique sur chaque item pour voir les détails et un exemple.

A01
Broken Access Control
Contrôle d'accès défaillant — #1 depuis 2021

Un utilisateur peut accéder à des ressources ou effectuer des actions non autorisées. Exemple : l'URL /admin/users accessible sans être admin.

GET /api/users/123/profile ← Tu vois ton profil
GET /api/users/124/profile ← Tu vois le profil d'un autre !

✅ Fix : Vérifier les autorisations côté serveur pour CHAQUE requête.

A02
Cryptographic Failures
Mauvais chiffrement des données sensibles

Données sensibles transmises en HTTP (pas HTTPS), mots de passe stockés en MD5, clés de chiffrement hardcodées dans le code source.

password_hash = md5(password) ← CASSÉ !
password_hash = bcrypt(password, 12) ← Correct
A03
Injection (SQLi, XSS, LDAP...)
Code malveillant injecté dans les entrées utilisateur

SQL Injection : données utilisateur intégrées directement dans une requête SQL sans validation.

Requête vulnérable :
SELECT * FROM users WHERE name='' OR '1'='1'
→ Retourne TOUS les utilisateurs !

Fix : requêtes préparées (parameterized queries)
A04
Insecure Design
Failles dans la conception même de l'application

La sécurité doit être pensée DÈS la conception, pas ajoutée après. Ex: système de reset de mot de passe sans limite de tentatives → force brute possible.

A05
Security Misconfiguration
Configuration par défaut non sécurisée

Mots de passe par défaut ("admin/admin"), pages d'erreur qui révèlent la stack technique, services inutiles exposés, permissions trop larges.

Error: MySQL server at 192.168.1.1:3306 — Version 5.7.2
← Révèle l'IP interne ET la version de la DB !
A06
Vulnerable Components
Bibliothèques et frameworks avec vulnérabilités connues

Utiliser des dépendances obsolètes avec des CVE connues. Log4Shell (CVE-2021-44228, score 10.0) a affecté des millions d'applications car Log4j était utilisé partout.

A07
Auth & Identification Failures
Authentification défaillante

Pas de MFA, mots de passe faibles acceptés, sessions non expirées, tokens JWT mal validés, absence de protection contre la force brute.

A08
Software & Data Integrity Failures
Mises à jour et pipelines CI/CD non sécurisés

Un attaquant compromet le pipeline de déploiement (supply chain attack) et injecte du code malveillant dans une mise à jour légitime. Ex: SolarWinds 2020.

A09
Security Logging Failures
Absence de logs et monitoring

Sans logs : impossible de détecter une intrusion, de comprendre ce qui s'est passé, ou de répondre à l'incident. Les logs doivent être horodatés (NTP !) et protégés.

A10
Server-Side Request Forgery (SSRF)
Forcer le serveur à faire des requêtes internes

L'attaquant force le serveur à faire une requête vers une ressource interne inaccessible depuis l'extérieur. Ex: accès aux métadonnées AWS (169.254.169.254).

POST /fetch?url=http://169.254.169.254/latest/meta-data/
→ Retourne les credentials AWS du serveur !

Types de Malwares

🔐
Ransomware

Chiffre tes fichiers et demande une rançon. Attaque la disponibilité. Propagation : phishing, exploits, RDP exposé.

🕵️
Spyware

Collecte des informations à ton insu (frappes clavier, captures écran, historique web). Attaque la confidentialité.

🐴
Trojan (Cheval de Troie)

Se présente comme un logiciel légitime mais contient une porte dérobée. Contrairement à un virus, il ne se réplique pas.

🪱
Worm (Ver)

Se réplique automatiquement d'une machine à l'autre sans action humaine. WannaCry utilisait EternalBlue pour se propager sur tout le réseau en minutes.

🔗
Botnet

Réseau de machines infectées (zombies) contrôlées par un Command & Control (C2). Utilisé pour DDoS, spam, minage de crypto, vol de données.

🥷
Rootkit

Se cache dans le système en modifiant l'OS pour devenir invisible. Très difficile à détecter et supprimer. Nécessite souvent une réinstallation complète.

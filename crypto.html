C'est quoi la Cryptographie ?

La cryptographie, c'est la science de la communication secrète. Elle permet de transformer un message lisible en un message incompréhensible pour toute personne non autorisée. Elle existe depuis 4000 ans !

🔐 Cryptographie

Transformation d'un message en utilisant une clé. Si tu as la clé → tu peux lire. Sans la clé → impossible. C'est réversible avec la bonne clé.

Texte clair + Clé = Texte chiffré
ex: "BONJOUR" + clé 3 = "ERQMRXU"
🖼️ Stéganographie

Cacher l'existence même du message — pas le chiffrer. Exemple : cacher du texte dans les pixels d'une image. Personne ne sait qu'il y a un message caché.

Cryptographie : "Je sais qu'il y a
un message, mais je ne peux pas le lire"
Stéganographie : "Je ne sais même pas
qu'il y a un message ici !"
📝 Plain Text (Texte en clair)

Le message original, lisible par n'importe qui. Aussi appelé "cleartext" ou "plaintext". C'est ce qu'on veut protéger.

🔒 Cipher Text (Texte chiffré)

Le message après chiffrement. Incompréhensible sans la clé de déchiffrement. Même en l'interceptant, on ne peut rien faire.

Jules César décalait chaque lettre de 3 positions dans l'alphabet pour ses messages militaires. C'est la plus simple des substitutions monoalphabétiques. Essaie ci-dessous !

Crypto classique

Substitution vs Transposition

🔄 Substitution

Chaque lettre est remplacée par une autre lettre ou symbole. L'ordre reste le même, mais les lettres changent.

César (décalage 3) :
A → D, B → E, C → F...

ROT13 (décalage 13) :
BONJOUR → OBAWHBE

Vigenère (clé = "KEY") :
HELLO + KEYKE → RIJVS
🔀 Transposition

Les lettres restent les mêmes mais leur ordre est mélangé selon une clé.

Transposition en colonnes :
ATTAQUEZDEMAIN
Clé : 3 4 1 2
Rangée 1 : A T T A
Rangée 2 : Q U E Z
Rangée 3 : D E M A
Rangée 4 : I N . .
→ Colonnes dans ordre clé 1,2,3,4...

L'opérateur XOR — La magie du chiffrement

XOR (Exclusive OR) est l'opération binaire au cœur de presque tous les algorithmes de chiffrement modernes (AES, RC4, OTP). Sa propriété magique : A XOR B XOR B = A — donc chiffrer puis rechiffrer avec la même clé donne le texte original !

🔑 One-Time Pad (OTP) — Indéchiffrable

Si la clé XOR est :

  • ✅ Aussi longue que le message
  • ✅ Vraiment aléatoire
  • ✅ Utilisée une seule fois

→ Indéchiffrable mathématiquement prouvé !

Problème : distribuer une clé aussi longue que le message de façon sécurisée est impossible à grande échelle.

🔄 Mode CTR (Counter)

AES-CTR : un compteur est chiffré, puis XORé avec le texte clair. Permet de chiffrer des données en streaming. C1 = XOR(P1, AES(clé, compteur1)).

🔑 Symétrique — 1 clé

Modes de fonctionnement

Par blocs (block cipher) : AES — divise le message en blocs de 128 bits, chiffre bloc par bloc.
Par flux (stream cipher) : RC4, ChaCha20 — chiffre bit par bit / octet par octet. Plus rapide pour les flux continus.
🔐 Asymétrique — 2 clés

Une paire de clés : clé publique (partagée librement) et clé privée (secrète). Résout le problème de l'échange de clés ! Plus lent que le symétrique.

Pour chiffrer vers Alice : utilise sa clé publique. Seule Alice peut déchiffrer avec sa clé privée.

💡 En pratique : chiffrement hybride (HTTPS/TLS)

Dans HTTPS, on utilise les deux ! L'asymétrique pour échanger une clé de session, puis le symétrique pour chiffrer les données (AES est 1000x plus rapide que RSA).

1. Alice génère une clé symétrique aléatoire (session key)
2. Elle la chiffre avec la clé publique RSA du serveur
3. Le serveur déchiffre avec sa clé privée RSA → obtient la session key
4. Toute la communication se fait avec AES (symétrique) — rapide !
Crypto — Intégrité

Fonctions de Hachage

Une fonction de hachage transforme n'importe quelle donnée en une empreinte de taille fixe. C'est à sens unique (irréversible), déterministe (même entrée → même sortie), et l'effet avalanche garantit qu'un seul bit changé → empreinte totalement différente.

MD5 (128 bits — CASSÉ, ne plus utiliser !)
SHA-256 (256 bits — Recommandé)
⚡ Propriétés d'une bonne fonction de hachage
  • ✅ Déterministe — même entrée → même hash
  • ✅ Irréversible — impossible de retrouver l'original
  • ✅ Effet avalanche — 1 bit changé → hash totalement différent
  • ✅ Résistance aux collisions — impossible de trouver 2 entrées avec le même hash
  • ✅ Rapide à calculer
📋 Usages des fonctions de hachage
  • 🔐 Mots de passe — stockés en hash, jamais en clair
  • ✅ Intégrité fichiers — vérifier qu'un fichier n'a pas été modifié
  • ⛓️ Blockchain — chaque bloc contient le hash du précédent
  • 🖊️ Signatures numériques — on signe le hash, pas le document entier
  • 🔍 Détection de malwares — base de signatures antivirales
Attaques sur la cryptographie

Cryptanalyse — Comment casser un chiffrement

💪 Force Brute

Essayer toutes les clés possibles. Pour AES-256 : 2²⁵⁶ possibilités. L'âge de l'univers ne suffirait pas avec tous les supercalculateurs du monde.

📊 Analyse de Fréquence

Exploite la distribution des lettres dans une langue. En français, E (17%), A (8%), I (7%)... Casse facilement César et Vigenère.

📖 Dictionnaire

Tester une liste de mots communs comme clés/mots de passe. 90% des mots de passe sont dans les 10 000 premiers mots de passe les plus utilisés.

🌈 Rainbow Tables

Table précalculée hash→texte. Permet de "retrouver" un mot de passe depuis son hash en quelques secondes. Contre-mesure : salage (salt).

🔮 Collision

Trouver deux entrées différentes avec le même hash. MD5 est cassé (collisions trouvées en 2004). SHA-1 aussi. SHA-256 est sûr.

🔑 Padding Oracle

Attaque sur les algorithmes par blocs. Le bourrage (padding) est utilisé pour compléter le dernier bloc à la taille requise (ex: blocs de 128 bits avec AES).

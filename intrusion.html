Module 02

Gestion des Intrusions

Détecter, analyser et répondre aux tentatives d'attaques. IDS, IPS, CVE, Zero-Day, pare-feux, DMZ — tout ce qu'il faut pour défendre une infrastructure.

Terminologie essentielle

Intrusion vs Incident

🚪 Intrusion = La tentative

Quelqu'un essaie d'accéder sans autorisation. Il n'a pas encore réussi. On peut encore le bloquer. L'IDS/IPS surveille et détecte ces tentatives.

Exemples d'intrusion :
→ Scan de ports (nmap)
→ Tentative de connexion SSH par force brute
→ Envoi d'un payload malveillant
→ Exploit non réussi
💥 Incident = La conséquence

L'attaquant a réussi. Il y a eu un impact sur la confidentialité, l'intégrité ou la disponibilité. On entre en mode réponse aux incidents.

Exemples d'incident :
→ Serveur compromis
→ Fuite de données
→ Ransomware déployé
→ Site web défiguré
Concept fondamental

Cycle de vie d'une Vulnérabilité

Ce schéma explique comment une faille naît, est découverte, exploitée et finalement corrigée. Ton prof a dit que c'était TRÈS important.

Phase 0
La faille existe (silencieusement)
Un développeur écrit du code avec un bug. La faille existe mais personne ne le sait encore — ni le vendeur, ni les hackers.
Phase 1 — ZERO DAY
🔴 Zero-Day Vulnerability découverte
Un chercheur (ou un hacker) découvre la faille. Le vendeur a eu 0 jour pour la corriger. C'est la phase la plus dangereuse — le vendeur ne sait pas encore.
Phase 2 — ZERO DAY
🔴 Zero-Day Attack
Des attaquants exploitent la faille avant qu'elle soit connue. Pas de défense possible basée sur les signatures — l'IDS ne peut pas la détecter. Seule l'analyse comportementale peut aider.
Phase 3 — CVE
🟠 Publication CVE
La faille est officiellement enregistrée dans la base CVE (cve.org). Elle reçoit un identifiant unique CVE-ANNÉE-NUMÉRO et un score CVSS. Le monde entier est maintenant informé.
Phase 4
🟡 Développement du patch
Le vendeur développe un correctif. Entre la publication CVE et la disponibilité du patch, les systèmes restent vulnérables. C'est la fenêtre d'exposition.
Phase 5
✅ Patch disponible
Le correctif est publié. Les admins système doivent l'appliquer le plus vite possible. Les systèmes non patchés restent vulnérables indéfiniment.
📋 Format CVE — Comment le lire
CVE-2021-44228 ← Log4Shell (Apache Log4j)
CVE = Common Vulnerabilities and Exposures
2021 = Année de publication
44228 = Numéro séquentiel unique

CVSS Score : 10.0 (CRITIQUE)
Impact : Exécution de code à distance sans authentification
Détection d'intrusions

IDS vs IPS

👁️ IDS — Intrusion Detection System

Passif — observe et alerte. Ne bloque rien. Placé en dehors du flux réseau (mode "miroir").

  • ✅ Détecte les intrusions
  • ✅ Génère des alertes
  • ✅ Ne peut pas être bloqué par l'attaquant
  • ❌ Ne bloque pas les attaques
  • ❌ Faux positifs possibles
NIDS (réseau)
HIDS (hôte)
Snort
Suricata
🛡️ IPS — Intrusion Prevention System

Actif — détecte ET bloque. Placé en ligne dans le flux réseau. Peut réinitialiser les connexions suspectes.

  • ✅ Détecte ET bloque en temps réel
  • ✅ Peut modifier le trafic
  • ❌ Peut devenir un point de défaillance
  • ❌ Faux positifs → blocage légitime
  • ❌ Latence ajoutée au trafic
NIPS
HIPS
EDR
UTM
⚠️ Faux Positifs vs Faux Négatifs
Faux Positif (False Positive)
Le système alerte pour une activité légitime. Exemple : un scan de ports fait par l'admin génère une alerte. Trop de faux positifs → les analystes ignorent les alertes (alert fatigue).
Faux Négatif (False Negative)
Une vraie attaque passe sans déclencher d'alerte. C'est le scénario le pire — on croit être protégé mais on ne l'est pas.
Architecture de sécurité

Pare-feux & Zones de sécurité

🔥 Types de pare-feux
  • L3 — Filtrage de paquets : examine IPs et ports. Rapide mais basique.
  • L4 — Stateful : suit l'état des connexions TCP. Plus intelligent.
  • L7 — Proxy applicatif : inspecte le contenu applicatif (HTTP, DNS). Le plus sécurisé.
  • WAF : spécialisé pour les apps web — bloque XSS, SQLi, CSRF.
  • NGFW (Next-Gen) : combine tout + ML/IA pour détection avancée.
🔍 DPI — Deep Packet Inspection

Le pare-feu L7 utilise le DPI pour inspecter le contenu des paquets (pas juste les en-têtes). Il peut lire le contenu HTTP, détecter des malwares dans des fichiers, bloquer des protocoles spécifiques.

Catalogue des attaques

Types d'Attaques réseau

🌊
DDoS — Déni de Service Distribué
Des milliers de machines zombies (botnet) inondent un serveur de requêtes. Il sature et tombe. Les 3 modes : volumétrique, protocole, applicatif.
🛡️ Mitigation : Cloudflare, scrubbing center, rate limiting, anycast
🕵️
MITM — Man In The Middle
L'attaquant s'intercale entre deux parties pour lire et/ou modifier les échanges. Possible sur WiFi non chiffré, ARP poisoning, DNS spoofing.
🛡️ Mitigation : HTTPS, certificats, VPN, HSTS
🎣
Phishing / Spoofing
Usurpation d'identité pour tromper l'utilisateur. Email qui imite sa banque, faux site web, faux numéro de téléphone. Vecteur d'attaque n°1.
🛡️ Mitigation : SPF, DKIM, DMARC, formation utilisateurs
💀
Ransomware
Malware qui chiffre tous tes fichiers et demande une rançon. Attaque la disponibilité. Se propage via phishing, vulnérabilités non patchées.
🛡️ Mitigation : Sauvegardes 3-2-1, EDR, mises à jour
🔓
CAM Table Flooding
Remplissage de la table MAC du switch jusqu'à saturation. Le switch se comporte comme un hub et diffuse tous les paquets → écoute possible.
🛡️ Mitigation : Port security sur les switches, limitation MAC
⚗️
SQL Injection
Injection de commandes SQL dans un formulaire non sécurisé. Permet de lire, modifier ou supprimer des données. Ex: ' OR '1'='1
🛡️ Mitigation : Requêtes préparées, WAF, validation des entrées
